# 09 全量测试方案与用例清单（安卓端上线标准）

## 1. 测试目标

确保 EvoMind 在国内安卓主流机型下功能完整、性能稳定、合规可上架。

## 2. 测试范围

- 功能测试：账号、引导、信息源、认知、内化、行动、订阅、支付、导出、注销
- 接口测试：成功、失败、幂等、重试、超时
- 性能测试：启动、滑动帧率、内存、耗电、网络耗时
- 兼容测试：Android 10-14，华为/小米/OPPO/vivo/荣耀
- 安全测试：鉴权、注入、重放、越权、敏感信息泄露
- 合规测试：AI标记、内容审核、隐私权限触发与说明

## 4. 自动化测试分层

- 单元测试：ViewModel、UseCase、Repository
- 集成测试：Room + 网络仓库 + 同步任务
- UI 自动化：登录、导入、卡片阅读、支付回调兜底
- 接口自动化：Postman/Newman 或 JMeter 场景

## 5. 性能测试基线

- 启动时间：≤3 秒
- 首屏渲染：≤1.5 秒
- 卡片切换掉帧：< 3%
- OCR 单张识别：≤3 秒
- 内化重任务：≤5 秒（Agent 训练除外）
- 常驻内存：≤500MB

## 6. 缺陷分级

- P0：阻断主流程或支付不可用，必须当日修复
- P1：影响核心体验，48 小时内修复
- P2：一般体验问题，版本内修复
- P3：优化建议，排入迭代
